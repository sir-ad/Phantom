# Page Spec: Docs

Owner: PhantomPM Web Team  
Last Updated: 2026-02-13  
Status: Planned

## Goal

Provide fast path from question to answer with clear trust boundaries.

## IA Requirements

1. top-level sections: Status, User, Technical, PM, Vision
2. persistent left navigation
3. searchable content index
4. capability status explanation visible in docs landing

## UX Requirements

1. every user-facing capability doc shows status tags
2. quick links for install, quickstart, troubleshooting
3. clear separation between vision content and factual docs

## Acceptance Criteria

1. docs users reach target answer in <= 3 clicks for top workflows
2. status docs are discoverable from docs landing and homepage
