# Website PRD: phantom.pm

Owner: PhantomPM Web + PM Team  
Last Updated: 2026-02-13  
Status: Planned

## Product Goal

Build a launch-ready website that converts qualified visitors into successful PHANTOM installations.

## Primary Objectives

1. Make one-line installation the dominant CTA.
2. Communicate value quickly with credible proof.
3. Keep all capability claims aligned with status model.
4. Route users into docs and first-use success path.

## Target Audience

1. product-minded developers
2. startup PMs
3. solo founders
4. product engineers and team leads

## Core User Journey

1. land on homepage
2. understand value proposition in <20 seconds
3. copy install command
4. complete install
5. follow quickstart to first artifact

## Information Architecture

1. `/` homepage
2. `/install`
3. `/docs`
4. `/modules`
5. `/integrations`
6. `/security`
7. `/roadmap`
8. `/changelog`

## Functional Requirements

1. install command component with copy action
2. status-tagged feature sections
3. link to capability matrix and demo-vs-real docs
4. conversion instrumentation for install flow

## Non-Functional Requirements

1. fast load and responsive layout
2. accessible content and keyboard interactions
3. SEO and metadata hygiene
4. privacy-conscious analytics

## Acceptance Criteria

1. install CTA path measurable end-to-end
2. no untagged feature claims
3. docs links present from key conversion pages
